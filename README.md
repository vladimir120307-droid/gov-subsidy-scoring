# Система меритократического скоринга сельскохозяйственных субсидий

## Название и описание

**Gov Subsidy Scoring** -- веб-приложение для перехода от системы распределения субсидий "кто первый подал заявку" к меритократическому скорингу сельхозпроизводителей. Разработано для Министерства сельского хозяйства Республики Казахстан в рамках хакатона Decentrathon 5.0 (Gov Case).

Система анализирует реестр из 36 652 заявок на субсидии в сфере животноводства за 2025 год, формирует профиль каждого из ~2 900 производителей по 14 факторам и выстраивает рейтинг на основе комбинированной модели (60% ML + 40% экспертные правила).

---

## Проблема

Текущая система распределения государственных субсидий в животноводстве основана на принципе "первым пришёл -- первым получил" (FCFS, First-Come-First-Served). Это приводит к ряду проблем:

- **Отсутствие учёта эффективности**: субсидии получают производители, которые быстрее подали заявку, а не те, кто эффективнее осваивает средства.
- **Неравный доступ**: производители из удалённых регионов физически не могут подать заявку раньше городских.
- **Низкое освоение средств**: часть субсидий уходит производителям, которые затем не осваивают выделенные средства.
- **Непрозрачность**: нет объективных критериев приоритизации.
- **Отсутствие обратной связи**: производители не понимают, что нужно улучшить для получения субсидий.

Наша система решает эти проблемы, заменяя очередь на объективный рейтинг по 14 факторам.

---

## Решение

Меритократическая система скоринга, которая:

1. **Ранжирует** производителей по объективным критериям (история одобрений, освоение средств, диверсификация, и др.).
2. **Объясняет** каждый балл -- производитель видит свои сильные и слабые стороны.
3. **Обеспечивает справедливость** -- встроенный анализ предвзятости по регионам и направлениям (Gini, Kruskal-Wallis, Lorenz).
4. **Позволяет настройку** -- комиссия МСХ может менять веса факторов (human-in-the-loop).
5. **Формирует шортлисты** с экспортом в CSV для интеграции в процесс принятия решений.

## Архитектура модели

### Комбинированный подход: 60% ML + 40% правила

Система использует ансамбль из двух моделей:

**1. ML-модель (GradientBoostingRegressor, вес 60%)**

- Алгоритм: Gradient Boosting Regression (scikit-learn)
- Гиперпараметры: 200 деревьев, глубина 5, learning_rate=0.1, subsample=0.8
- Целевая переменная: синтетическая, формируется из комбинации метрик:
  - completion_rate * 0.35
  - approval_rate * 0.25
  - utilization_rate * 0.20
  - direction_diversity * 0.10
  - (1 - rejection_rate) * 0.10
- Выход: нормализованный балл [0, 100]

**2. Правиловая модель (вес 40%)**

- Каждый из 14 признаков нормализуется в [0, 1] через MinMaxScaler
- Применяются экспертные веса (положительные и отрицательные)
- Итог: взвешенная сумма, нормализованная в [0, 100]
- Полная прозрачность: каждый фактор имеет известный вклад

**Обоснование гибридного подхода:**

- ML ловит нелинейные паттерны, которые эксперт может пропустить
- Правила обеспечивают прозрачность и объяснимость, критичную для государственного сектора
- Баланс 60/40 настраиваем комиссией через интерфейс (human-in-the-loop)
- При 100% ML теряется интерпретируемость; при 100% правил -- точность

**Итоговый балл:**

```
combined_score = ml_weight * ml_score + rule_weight * rule_score
```

## Валидация модели

### Кросс-валидация

- Метод: 5-fold Cross-Validation
- Метрика: R-squared (коэффициент детерминации)
- Результат: R2 = 0.9992 (среднее по 5 фолдам)

### Дополнительные метрики

- MAE (средняя абсолютная ошибка): вычисляется на каждом фолде
- RMSE (среднеквадратичная ошибка): вычисляется на каждом фолде
- Пермутационная важность: подтверждает стабильность ранжирования признаков

### Baseline-сравнение

Сравнение с системой FCFS (случайный отбор топ-100):

| Метрика                 | FCFS (текущая) | Скоринг (наша) | Прирост  |
|------------------------|----------------|----------------|----------|
| Средний балл (топ-100) | ~50            | ~85            | +70%     |
| Среднее одобрение      | ~65%           | ~95%           | +46%     |
| Среднее освоение       | ~40%           | ~80%           | +100%    |

## Справедливость

Встроенный модуль анализа справедливости проверяет отсутствие систематической предвзятости:

### Коэффициент Джини

- Gini (баллы): оценка неравенства распределения баллов между производителями
- Gini (суммы): оценка неравенства распределения запрошенных сумм
- Значение < 0.3 -- умеренное неравенство (норма)

### Тест Краскела-Уоллиса

- Непараметрический тест на однородность распределения баллов между регионами
- H-статистика и p-value
- p > 0.05 -- нет статистически значимых различий между регионами

### Кривая Лоренца

- Визуальное сопоставление реального распределения с идеальным равенством
- Строится для баллов и для сумм субсидий

### Региональный анализ

- Отклонение среднего балла региона от общего среднего (в sigma)
- Флаг предвзятости при отклонении > 1 sigma
- Коэффициент вариации между регионами

## Объяснимость (Explainability)

Система обеспечивает полную прозрачность оценки:

1. **Вклад каждого фактора**: для каждого производителя показан вклад каждого из 14 факторов в итоговый балл (в баллах).
2. **Сильные и слабые стороны**: автоматическое определение топ-5 сильных сторон и топ-3 слабых.
3. **Рекомендации**: текстовая рекомендация (приоритетное субсидирование / стандартный порядок / дополнительная проверка / не рекомендован).
4. **Важность признаков ML**: встроенная feature importance из GradientBoosting + пермутационная важность.
5. **Процентильный ранг**: позиция производителя относительно всех остальных.
6. **Рекомендации по улучшению**: конкретные действия для повышения балла.

## Ограничения

1. **Cold-start проблема**: новые производители без истории заявок не могут получить корректный скоринг. Система опирается на накопленную статистику.
2. **Данные за один год**: модель обучена на данных 2025 года. Сезонные и годовые тренды могут измениться.
3. **Только животноводство**: текущая модель адаптирована под направления животноводства (скотоводство, овцеводство, птицеводство и др.). Для растениеводства потребуется адаптация.
4. **Синтетическая целевая переменная**: ML-модель обучена на комбинации метрик, а не на внешней независимой оценке качества производителя.
5. **Нет учёта внешних факторов**: погодные условия, эпизоотическая обстановка, рыночные цены -- не учитываются.
6. **Качество исходных данных**: система зависит от полноты и корректности данных ИСС. Пропуски и ошибки в реестре влияют на скоринг.
7. **Статический анализ**: скоринг вычисляется на момент запуска. Для промышленной эксплуатации нужен механизм регулярного обновления.
8. **Отсутствие A/B-тестирования**: эффективность скоринговой системы по сравнению с FCFS не проверена в реальных условиях.
9. **Producer ID -- приближение**: ID производителя извлекается из первых 11 цифр номера заявки, что может приводить к неточностям группировки.
10. **Региональные различия**: социально-экономические различия между регионами (уровень развития инфраструктуры, доступ к интернету) могут создавать систематические смещения.

## Приватность и безопасность

- Все данные обрабатываются **локально** на сервере приложения.
- Система **не отправляет данные** во внешние API или облачные сервисы.
- Данные сессии **не сохраняются** между запусками.
- ID производителей могут быть **анонимизированы** (используются только для группировки).
- Приложение не требует аутентификации и работает в изолированной среде.
- Исходные данные реестра ИСС используются только для аналитических целей.
- Рекомендуется разворачивать систему в защищённом контуре МСХ.
